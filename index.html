Docker & Kubernetes

Conceptos clave y comparación esencial para desarrollo y despliegue

Resumen

Docker empaqueta aplicaciones y dependencias en contenedores ligeros. Kubernetes orquesta y escala esos contenedores en clústeres de nodos.

Datos relevantes

Tema Docker Kubernetes
Propósito Construcción, empaquetado y ejecución de contenedores Orquestación: despliegue, escalado, auto-reparación
Componentes clave Imagen, Dockerfile, Contenedor, Volumen Pod, Deployment, Service, Ingress, Namespace
Casos de uso Dev local, CI/CD, microservicios simples Alta disponibilidad, balanceo, rolling updates
Escalabilidad Manual (docker compose replicas) Automática (HPA), clúster multi-nodo
Comandos típicos docker build/run/logs kubectl apply/get/describe

Flujo básico con Docker

  1. Escribe un Dockerfile.
  2. docker build -t miapp:1.0 .
  3. docker run -p 8080:80 miapp:1.0

Despliegue básico en K8s

  1. Define deployment.yaml y service.yaml.
  2. kubectl apply -f deployment.yaml
  3. kubectl get pods,svc
Consejo: usa Docker para empaquetar y probar rápidamente; pasa a Kubernetes cuando necesites resiliencia, escalado y actualizaciones sin caída.
